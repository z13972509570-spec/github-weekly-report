# 📊 GitHub Weekly Report

> 自动统计 GitHub 仓库代码改动、识别风险点、生成周报并推送

## ✨ 功能

- 📈 **代码改动统计** — 提交次数、代码行数、文件变更
- 🔍 **风险识别** — 安全漏洞、依赖问题、代码质量问题
- 📋 **详细报告** — Markdown + JSON 双格式输出
- 💬 **多渠道推送** — 微信/邮件/Slack
- ⏰ **定时任务** — 每周自动生成报告

## 🚀 快速开始

```bash
pip install -e .

# 生成周报
weekly-report generate

# 推送到微信
weekly-report push
```

## 📊 报告内容

### 1. 代码改动统计
- 提交数量
- 代码行数变化
- 文件变更统计

### 2. 风险清单
- 🔴 高风险：安全漏洞、重大bug
- 🟠 中风险：依赖过时、性能问题
- 🟡 低风险：代码规范、小问题

### 3. 审查重点
- 建议优先关注的变更
- 代码质量分析
- 建议改进项
